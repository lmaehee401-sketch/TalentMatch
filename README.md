# TalentMatch

**AI 驱动的招聘效率工具** —— 语义级简历匹配、批量候选人排序、招聘漏斗诊断，三合一。

🔗 在线体验：https://lmaehee401-sketch.github.io/TalentMatch/
📄 完整产品文档：[PRD.md](./PRD.md)

---

## 这是什么

传统关键词筛选简历，识别不了「用分布式锁解决超发问题」等价于「有分布式系统设计能力」这类语义表达，也容易在看了几十份简历后评判标准发生漂移。TalentMatch 用大语言模型做语义级的能力画像匹配，并支持 HR 对匹配结果反馈、AI 据此调整画像重新匹配——**不是一次性打分，而是人机协作的迭代过程**。

工具定位为**半自动化的辅助**：负责结构化初筛、证据呈现、瓶颈定位，候选人的综合判断仍需 HR 通过面试完成，最终决策权始终在人。

## 三大功能

| 模块 | 能做什么 |
|---|---|
| 📋 **单份匹配** | 输入 JD + 简历，四步流水线生成评分、硬性条件核验表、语义匹配详情（含原文证据）、STAR 面试追问指南 |
| 📊 **批量对比** | 支持粘贴 / 多文件上传 / ZIP 导入（单次上限 20 份），自动排行榜，点击展开详情 |
| 📈 **漏斗诊断** | 输入招聘各环节数据，AI 对比行业基准定位瓶颈环节，给出改进建议 |

单份匹配与批量对比均支持**反馈迭代**：对结果不满意时，用标签（太严格 / 太宽松 / 补硬性条件等）或自由文本告诉 AI，它会重构画像并重新匹配，多轮迭代历史可追溯。

## 技术实现

- **纯前端**：HTML + CSS + JS，零框架依赖，约 1600 行，部署于 GitHub Pages
- **AI 引擎**：DeepSeek Chat API，用户自备 Key，Key 与简历数据均不经过任何中间服务器
- **本地存储**：localStorage + Cookie 双存储，互为兜底，跨标签页共享
- **健壮性**：指数退避重试（503/429 自动重试 3 次）、文件导入逐项容错、ZIP 智能过滤系统文件

## 快速开始

1. 打开 [在线地址](https://lmaehee401-sketch.github.io/TalentMatch/)
2. 准备一个 [DeepSeek API Key](https://platform.deepseek.com/)（新用户有 500 万免费 token，够用 500+ 次匹配）
3. 粘贴 JD 和简历，即可开始分析

本地运行只需一个静态文件服务器，例如：

```bash
git clone https://github.com/lmaehee401-sketch/TalentMatch.git
cd TalentMatch
python -m http.server 8000
# 打开 http://localhost:8000
```
